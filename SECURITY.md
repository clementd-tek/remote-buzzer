# Security policy

## Supported versions

Only the latest version on `main` receives security fixes.

## Reporting a vulnerability

**Please do not open a public issue for security vulnerabilities.**

Report them privately via GitHub's [Security Advisories](../../security/advisories/new) feature. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested fix if you have one

You can expect an acknowledgement within 48 hours and a resolution or status update within 7 days.

## Scope

This project is designed for **local network / LAN use**. It does not implement authentication. Deploying it on the public internet without a firewall, VPN, or additional access control is out of scope for security support — do so at your own risk.
